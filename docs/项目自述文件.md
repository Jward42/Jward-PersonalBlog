个人网站与 MLE 作品集

1. 项目目标

本项目旨在创建一个现代化的个人网站，主要有两个目的：

个人爱好： 作为个人博客和摄影作品的记录与分享平台。

职业展示 (MLE)： 作为机器学习工程师 (MLE) 的专业作品集，用于展示项目经历、技术博客和工程能力，辅助求职。

2. 核心功能 (Features)

主页 (Homepage):

个人简介 (Bio) 和照片。

内容“抓手” (Hooks)：展示最新的博客文章和精选的摄影作品。

博客 (Blog):

支持 Markdown 写作。

文章按标签 (Tags) 分类和筛选。

用于发布技术（MLE 项目）和生活相关的文章。

摄影 (Photography):

瀑布流或网格画廊 (Gallery) 布局。

支持按标签 (Tags) 分类。

关于 (About) 页面:

详细的个人介绍、技能栈和简历。

联系 (Contact) 页面:

社交媒体链接或联系表单。

3. 架构 (Architecture)

本项目采用 "混合静态网站" (Hybrid Static Site) 架构。

核心理念： 网站本身是静态生成的（极快、安全、免费），但内容（特别是媒体文件）是动态管理的。

内容 (文本): 博客文章、页面内容等将作为本地的 Markdown (.md) 文件进行管理。

内容 (媒体): 所有图片（博客插图、摄影作品）将托管在云存储服务（如 Cloudinary, AWS S3）上，以保持 Git 仓库的轻量和高性能。

数据流： Markdown 文本通过 Git 管理，图片通过云存储管理。

4. 技术栈 (Tech Stack)

网站框架 (Framework): Astro

理由： 高性能（默认零 JS），对 Markdown 和内容驱动的网站有出色的支持。

样式 (Styling): Tailwind CSS

理由： 现代化的 Utility-First CSS 框架，能快速构建精美的 UI。

插件： 将使用 @tailwindcss/typography 插件，自动美化 Markdown 渲染后的 HTML。

文本管理 (Workflow): Git / GitHub

理由： 所有的 .md 文件将作为代码提交，利用 Git 进行版本控制。这是本项目的核心“后台”。

媒体管理 (Storage): Cloudinary (或同类云存储)

理由： 提供强大的图片 API 和慷慨的免费额度，用于存储和分发所有摄影作品和博客插图。

部署 (Deployment): Vercel (或 Netlify)

理由： 提供免费、无缝的 CI/CD（持续集成/持续部署）。能连接到 GitHub 仓库，实现 git push 自动构建和发布网站。

5. 项目意义 (作为 MLE 作品集)

此项目本身也是一个关键的工程作品，它展示了：

现代 Web 开发能力： 熟练使用 Astro 和 Tailwind。

工程工作流 (DevOps)： 理解并实践了 Git + CI/CD (Vercel) 的自动化部署流程。

API 与服务集成： 能够将第三方服务 (Cloudinary) 集成到应用中。

成果展示能力： 搭建了一个平台，用于清晰、图文并茂地展示其他 MLE 项目。
